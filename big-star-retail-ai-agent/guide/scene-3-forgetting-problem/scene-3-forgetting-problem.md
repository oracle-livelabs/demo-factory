# Lab 3: Scene 3 The Forgetting Problem

## Introduction

This scene demonstrates a common support failure: customers repeat the same story across sessions. You then test a memory-aware path that continues the case without restarting the conversation.

Estimated Time: 10 minutes

### Objectives

In this lab, you will:
- Load the historical timeline.
- Trigger the memory-enabled response.
- Compare cycle time and customer effort.

## Task 1: Load Prior Sessions

1. Open **Scene 3: The Forgetting Problem**.
2. Click **Load Sessions**.
3. Read the timeline entries shown on screen.

Expected result:
- Multiple prior contacts appear.
- The same issue is repeated across sessions.
- The pattern shows high customer effort and low continuity.

## Task 2: Test the Memory-Aware Path

1. Click **Try With Memory**.
2. Review the new response block.
3. Compare this response with the earlier timeline behavior.

Expected result:
- The agent acknowledges prior history.
- It avoids asking the customer to repeat details.
- Resolution moves faster with less friction.

## Task 3: Why this matters?

The key message here is that continuity is a business capability, not a convenience. Repeated customer explanations increase effort, lower trust, and extend cycle times, while durable context lets each interaction pick up where the last one ended. Oracle AI Database 26ai supports this through a converged approach where history, memory context, and policy knowledge remain queryable together, allowing AI-agent workflows to preserve context across sessions and deliver more predictable outcomes across teams and channels.
