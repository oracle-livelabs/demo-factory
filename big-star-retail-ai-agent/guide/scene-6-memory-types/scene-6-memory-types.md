# Lab 5: Scene 6 Memory Types

## Introduction

This scene breaks memory into clear business categories so users can understand why the agent made a decision. The UI presents memory cards and a combined outcome summary.

Estimated Time: 10 minutes

### Objectives

In this lab, you will:
- Load all memory-type cards.
- Review what each type contributes.
- Understand how memory composition improves decision quality.

## Task 1: Load Memory Types

1. Open **Scene 6: The 4 Memory Types**.
2. Click **Load Memory Types**.
3. Review all cards shown in the grid.

Expected result:
- You see memory categories with examples.
- Each card explains how that memory type influenced the case.
- The view is easy to explain to non-technical stakeholders.

## Task 2: Interpret the Combined Story

1. Read each card from top to bottom.
2. Focus on the case-specific examples and the summary text.
3. Compare the “with memory” and “without memory” performance indicators.

Expected result:
- You can trace how different memory types work together.
- The case appears faster and more reliable when memory is complete.
- The summary supports business storytelling for leadership reviews.

## Task 3: Why this matters?

This scene explains why memory design is central to dependable AI operations. Different memory types carry different value, and when they are managed as disconnected components, consistency and explainability both degrade. Oracle AI Database 26ai is a strong fit because its converged model supports mixed data patterns under one governance framework, helping customer AI-agent workflows simplify architecture while improving retrieval consistency, maintainability, and confidence in decisions.
