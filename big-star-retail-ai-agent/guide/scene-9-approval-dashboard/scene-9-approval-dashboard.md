# Lab 8: Scene 9 Approval Dashboard

## Introduction

This scene gives managers a practical approval view with green, yellow, and red lanes. You can load the backlog, review risk tiering, and approve or reject pending items.

Estimated Time: 10 minutes

### Objectives

In this lab, you will:
- Load the dashboard lanes.
- Review pending actions by risk level.
- Execute a manager approval decision.

## Task 1: Load and Review the Dashboard

1. Open **Scene 9: Marcus's Dashboard**.
2. Click **Load Dashboard**.
3. Review counts and lane groupings.

Expected result:
- Green, yellow, and red lanes are shown.
- Pending items are clearly visible.
- Case details include action, customer context, and value.

## Task 2: Approve or Reject a Pending Item

1. Find an item with status **pending**.
2. Click **Approve** or **Reject**.
3. Watch for follow-up updates and reload behavior.

Expected result:
- Decision feedback appears quickly.
- The selected case status changes from pending.
- The dashboard refreshes with the updated queue state.

## Task 3: Why this matters?

The dashboard scenario demonstrates that human oversight depends on data coherence as much as user interface design. Risk-tiered queues only work when statuses, priorities, and outcomes stay synchronized in real time. Oracle AI Database 26ai's converged architecture helps maintain that synchronization on a single transactional backbone, so customer AI-agent workflows can improve manager responsiveness and reporting quality without reconciliation delays.
